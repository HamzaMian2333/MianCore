# Contributing Guidelines

These guidelines help interns work together cleanly and avoid breaking the main project.

## Branch Naming

Use one of these formats:

```text
feature/short-description
bugfix/short-description
docs/short-description
research/short-description
```

Examples:

```text
feature/inventory-dashboard
feature/csv-upload
docs/agent-safety-rules
research/twilio-integration
```

## Pull Requests

Every pull request should include:

- What was changed
- Screenshots if the UI changed
- Testing steps
- Any known issues or blockers

Do not push directly to `main`.

## Code Style

- Use clear file names.
- Use readable function names.
- Keep functions small when possible.
- Add comments only where the code may be confusing.
- Never commit API keys, passwords, or `.env` files.

## Review Process

1. Intern creates a branch.
2. Intern opens a pull request.
3. Team lead reviews the code.
4. Changes are requested if needed.
5. Pull request is merged after approval.
